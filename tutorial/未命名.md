没有加入任何群组为啥要设置缓存
没有联系人，直接返回空切片并缓存（防止缓存穿透）

这样做主要是为了防止 **“缓存穿透”（Cache Penetration）**。

### 什么是缓存穿透？

如果一个用户真的没有加入任何群组，数据库查询结果就是空的。如果你**不设置**缓存，那么：

1. 用户每次打开页面请求群组列表。
2. 程序检查 Redis：发现没这个 Key（Cache Miss）。
3. 程序去查数据库：发现确实没有记录。
4. 由于没记录，你没写缓存。
5. 下一次请求：依然重复上述步骤，**依然会击穿缓存直接打到数据库上**。

### 为什么要存一个空列表 

```
[]
```

？

当你把一个空列表 

```
[]
```

 存入 Redis 后：

1. 下次请求到来。
2. 程序检查 Redis：发现有值，是一个空的 JSON 数组 
    
    ```
    []
    ```
    
    。
3. 程序直接返回空列表，**不再查询数据库**。
